<span data-ttu-id="d991f-p102">La consulta de delta permite a las aplicaciones detectar entidades recién creadas, actualizadas o eliminadas sin realizar una operación de lectura completa del recurso de destino con cada solicitud. ## <a name="use-delta-query-to-track-changes-in-a-resource-collection"></a><span data-ttu-id="d991f-107">Usar la consulta de delta para realizar el seguimiento de los cambios en una colección de recursos</span><span class="sxs-lookup"><span data-stu-id="d991f-107">Use delta query to track changes in a resource collection</span></span>

<span data-ttu-id="d991f-108">El patrón de llamada típico es el siguiente:</span><span class="sxs-lookup"><span data-stu-id="d991f-108">The typical call pattern is as follows:</span></span>

1. <span data-ttu-id="d991f-109">La aplicación empieza con una llamada a una solicitud GET con la función delta en el recurso deseado.</span><span class="sxs-lookup"><span data-stu-id="d991f-109">The application begins by calling a GET request with the delta function on the desired resource.</span></span>
2. <span data-ttu-id="d991f-110">Microsoft Graph envía una respuesta que contiene el recurso solicitado y un [token de estado](#state-tokens).</span><span class="sxs-lookup"><span data-stu-id="d991f-110">Microsoft Graph sends a response containing the requested resource and a [state token](#state-tokens).</span></span>

     <span data-ttu-id="d991f-p103">a.  Si se devuelve una dirección URL `nextLink`, podría haber más páginas de datos para recuperar en la sesión.      <span data-ttu-id="d991f-p104">b.  Si se devuelve dirección URL `deltaLink`, no hay más datos sobre el estado existente del recurso para devolver. 3. <span data-ttu-id="d991f-p105">Cuando la aplicación necesita obtener información sobre los cambios en el recurso, realiza una nueva solicitud con la dirección URL `deltaLink` recibida en el paso 2. 4. <span data-ttu-id="d991f-119">Microsoft Graph devuelve una respuesta que describe los cambios en el recurso desde la solicitud anterior, y una dirección URL `nextLink` o `deltaLink`.</span><span class="sxs-lookup"><span data-stu-id="d991f-119">Microsoft Graph returns a response describing changes to the resource since the previous request, and either a `nextLink` URL or a `deltaLink` URL.</span></span>

><span data-ttu-id="d991f-120">**Nota:** Los recursos almacenados en Azure Active Directory (por ejemplo, usuarios y grupos) admiten los escenarios de "sincronizar desde ahora".</span><span class="sxs-lookup"><span data-stu-id="d991f-120">**Note:** Resources stored in Azure Active Directory (such as users and groups) support "sync from now" scenarios.</span></span> <span data-ttu-id="d991f-121">Esto le permite omitir los pasos 1 y 2 anteriores (si no le interesa recuperar el estado del recurso completo) y solicitar la última `deltaLink` en su lugar.</span><span class="sxs-lookup"><span data-stu-id="d991f-121">This allows you to skip steps 1 and 2 above (if you are not interested in retrieving the full state of the resource) and ask for the latest `deltaLink` instead.</span></span> <span data-ttu-id="d991f-122">Anexar `$deltaToken=latest` a la función `delta` y la respuesta contendrá un `deltaLink` sin datos de recursos.</span><span class="sxs-lookup"><span data-stu-id="d991f-122">Append `$deltaToken=latest` to the `delta` function and the response will contain a `deltaLink` and no resource data.</span></span>

The following details are what you need to know about them:</span></span>
