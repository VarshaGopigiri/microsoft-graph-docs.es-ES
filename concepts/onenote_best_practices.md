## <a name="use-select-to-select-the-minimum-set-of-properties-you-need"></a><span data-ttu-id="b979d-104">Use $select para seleccionar el conjunto mínimo de propiedades que necesita.</span><span class="sxs-lookup"><span data-stu-id="b979d-104">Use $select to select the minimum set of properties you need</span></span>
<span data-ttu-id="b979d-105">Cuando consulta un recurso (por ejemplo, las secciones dentro de un bloc de notas), realiza una solicitud similar a la siguiente.</span><span class="sxs-lookup"><span data-stu-id="b979d-105">When you query for a resource (for example, sections inside a notebook), you make a request similar to the following.</span></span>

```http
GET ~/notebooks/{id}/sections
```

<span data-ttu-id="b979d-106">Esto recupera todas las propiedades de las secciones.</span><span class="sxs-lookup"><span data-stu-id="b979d-106">This retrieves all the properties of the sections.</span></span> <span data-ttu-id="b979d-107">Pero puede que no necesite todas las propiedades.</span><span class="sxs-lookup"><span data-stu-id="b979d-107">However, you might not need all properties.</span></span> <span data-ttu-id="b979d-108">Puede usar el parámetro de consulta `$select` para que devuelva solo las propiedades que quiera, como se muestra en el siguiente ejemplo de consulta.</span><span class="sxs-lookup"><span data-stu-id="b979d-108">You can use the `$select` query parameter to return just the properties that you want, as shown in the following example.</span></span>

```http
GET ~/notebooks/{id}/sections?$select=id,displayName
```

<span data-ttu-id="b979d-109">El mismo enfoque se aplica a otras API de OneNote.</span><span class="sxs-lookup"><span data-stu-id="b979d-109">The same approach applies to other OneNote APIs.</span></span>

## <a name="use-expand-instead-of-making-multiple-api-calls"></a><span data-ttu-id="b979d-110">Use $expand en lugar de realizar varias llamadas a la API.</span><span class="sxs-lookup"><span data-stu-id="b979d-110">Use $expand instead of making multiple API calls</span></span>
<span data-ttu-id="b979d-111">Supongamos que quiere recuperar todos los grupos de sección, secciones y blocs de notas del usuario en una vista jerárquica.</span><span class="sxs-lookup"><span data-stu-id="b979d-111">Suppose you want to retrieve all of the user’s notebooks, sections, and section groups in a hierarchical view.</span></span> <span data-ttu-id="b979d-112">Pueden conseguirlo mediante el procedimiento siguiente:</span><span class="sxs-lookup"><span data-stu-id="b979d-112">You might accomplish that by doing the following:</span></span>

* <span data-ttu-id="b979d-113">Llame a `GET ~/notebooks` para obtener la lista de blocs de notas.</span><span class="sxs-lookup"><span data-stu-id="b979d-113">Call `GET ~/notebooks` to get the list of notebooks.</span></span>
* <span data-ttu-id="b979d-114">Para todos los blocs de notas recuperados, llame a `GET ~/notebooks/{notebookId}/sections` para recuperar la lista de secciones.</span><span class="sxs-lookup"><span data-stu-id="b979d-114">For every retrieved notebook, call `GET ~/notebooks/{notebookId}/sections` to retrieve the list of sections.</span></span>
* <span data-ttu-id="b979d-115">Para todos los blocs de notas recuperados, llame a `GET ~/notebooks/{notebookId}/sectionGroups` para recuperar la lista de grupos de secciones.</span><span class="sxs-lookup"><span data-stu-id="b979d-115">For every retrieved notebook, call `GET ~/notebooks/{notebookId}/sectionGroups` to retrieve the list of section groups.</span></span>
* <span data-ttu-id="b979d-116">Opcionalmente, itere recursivamente por los grupos de sección.</span><span class="sxs-lookup"><span data-stu-id="b979d-116">Optionally recursively iterate through section groups.</span></span>

<span data-ttu-id="b979d-117">Aunque esto funcionará (con algunos ciclos de ida y vuelta al servicios adicionales), un enfoque mejor es usar el parámetro de consulta `$expand`.</span><span class="sxs-lookup"><span data-stu-id="b979d-117">While this will work (with a few extra sequential roundtrips to the service), a better approach is to use the `$expand` query parameter.</span></span> 

```http
GET ~/notebooks?$expand=sections,sectionGroups($expand=sections)
```

<span data-ttu-id="b979d-118">Esto producirá los mismos resultados en un solo ciclo de ida y vuelta, con un mejor rendimiento.</span><span class="sxs-lookup"><span data-stu-id="b979d-118">This will yield the same results in one network roundtrip, with better performance.</span></span>

## <a name="when-getting-all-pages-for-a-user-do-so-for-each-section-separately"></a><span data-ttu-id="b979d-119">Cuando quiera obtener todas las páginas de un usuario, hágalo para cada sección por separado</span><span class="sxs-lookup"><span data-stu-id="b979d-119">When getting all pages for a user, do so for each section separately</span></span>

<span data-ttu-id="b979d-120">Aunque Microsoft Graph expone un punto de conexión para recuperar todas las páginas, esta no es la mejor forma de obtener todas las páginas a las que tiene acceso el usuario.</span><span class="sxs-lookup"><span data-stu-id="b979d-120">While Microsoft Graph exposes an endpoint to retrieve all pages, this isn't the best way to get all the pages the user has access to.</span></span> <span data-ttu-id="b979d-121">Cuando el usuario tiene demasiadas secciones, esto puede conducir a tiempos de espera o a un rendimiento incorrecto.</span><span class="sxs-lookup"><span data-stu-id="b979d-121">When the user has too many sections, this can lead to timeouts or bad performance.</span></span> <span data-ttu-id="b979d-122">Es mejor recorrer en iteración cada sección, y obtener las páginas para cada uno de ellos por separado.</span><span class="sxs-lookup"><span data-stu-id="b979d-122">It is better to iterate each section, getting pages for each one separately.</span></span>

<span data-ttu-id="b979d-123">Por ejemplo, en lugar de usar esta llamada (se pagina esta API, por lo que no podrá recuperar todas las páginas a la vez):</span><span class="sxs-lookup"><span data-stu-id="b979d-123">For example, instead of using this call (this API is paged, so you won't be able to fetch the pages all at once):</span></span>

<span data-ttu-id="b979d-124">Es mejor usar la siguiente llamada varias veces (especialmente si no necesita todas las secciones):</span><span class="sxs-lookup"><span data-stu-id="b979d-124">It is better to use the following call several times (especially if you don't need all sections):</span></span>

